1. Instalacin y carga de paquetes avanzados

Se utilizarn readr para la importacin del archivo .csv, tidytable y data.table para el procesamiento de datos, y ggplot2 para la visualizacin.

Para la instalacin de tidytable, instalamos primero devtools, ya que tidytable no est en CRAN.

# Lista de libreras de CRAN necesarias
cran_libraries <- c("readr", "data.table", "ggplot2", "rmarkdown")

# Comprobar e instalar libreras de CRAN
for (lib in cran_libraries) {
  if (!requireNamespace(lib, quietly = TRUE)) {
    install.packages(lib)
  }
}

# Comprobar si devtools est instalado, si no, instalarlo
if (!requireNamespace("devtools", quietly = TRUE)) {
  install.packages("devtools")
}

# Instalar la librera desde GitHub si no est instalada
if (!requireNamespace("tidytable", quietly = TRUE)) {
  devtools::install_github("markfairbanks/tidytable")

3. Limpieza de datos

Columna por columna, vemos los siguientes problemas y peculiaridades:

  • 1- Column1 (X): No sirve para nada, se puede obviar.
  • 2- Item_Number: En algunas filas tiene -, en otras un nmero identificatorio, y en otras un precio. Debido a su ambigedad, puede obviarse.
  • 3- Set_Name: Muchas filas con NA, pero el resto parece que se refiere al nombre del set de LEGO. Tratamos NA como nombre desconocido.
  • 4- Amazon_price: El formato no est normalizado y no permite estudiarlo bien. Sobran el y ocasional *. Tratamos- como NA, precio de Amazon desconocido. Adems, Hay muy pocos datos relevantes as que no nos sirve para el estudio.
  • 5- Year: Ao de salida del set.
  • 6- Pages: Solo contiene NA, puede obviarse.
  • 7- unique_pieces: Nmero de piezas nicas en cada set.
  • 8- Theme: Tema del set.
  • 9- Pieces: Nmero de piezas de cada set.
  • 10- Price: Solo contiene NA, puede obviarse.
  • 11- Ages: Edad recomendada del set, los desconocidos en vez de nombrarlos NA, los nombra Ages_NA.
  • 12- Minifigures: Nmero de minifiguras de cada se.
  • 13- Packaging: Material de la caja donde viene el set.
  • 14- Weight: El formato no est normalizado y no permite estudiarlo bien. Incluye una conversin a libras, que obviaremos.
  • 15- Availability: Dnde se puede conseguir el set.
  • 16- Size: Tamao del set.

Conversin tipos de variable y eliminacin de columnas:

  • 1- Column1 (X): Eliminar
  • 2- Item_number: Eliminar
  • 3- Set_name: Categrica
  • 4- Amazon_price: Eliminar
  • 5- Year: Categrica
  • 6- Pages: Eliminar
  • 7- unique_pieces: Numrica
  • 8- Theme: Categrica
  • 9- Pieces: Numrica
  • 10- Price: Eliminar
  • 11- Ages: Categrica
  • 12- Minifigures: Numrica
  • 13- Packaging: Categrica
  • 14- Weight: Numrica
  • 15- Availability: Categrica
  • 16- Size: Categrica

4. Anlisis de datos

Una vez limpia nuestra fuente de datos, podemos empezar a analizar.

En este anlisis, no podremos estudiar el precio de los sets, ya que las filas que contenan precio (tanto en la columna Item_Number como la Amazon_Price) no iban asociadas a nada ms, el resto de columnas no contenan datos relevantes.Por lo tanto, el anlisis se har resolviendo a otro tipo de preguntas no monetarias:

  • Distribucin de sets por ao
  • Cantidad de sets por temtica
  • Tipos de empaques ms comunes
  • Nmero de piezas por set
  • Relacin nmero de piezas con nmero de minifiguras
  • Edad del pblico de LEGO y su relacin con el nmero de piezas
  • Relacin peso con tamao
  • Disponibilidad de los sets
